# 사이클 1: 테마 + 사용자 예약

## 한 줄 요약

규칙을 적용하면서 사용자가 직접 예약하는 기능을 만든다.

## 실습 환경

- `spring-roomescape-member` 저장소를 기반으로 진행한다.
- 온라인 코드 리뷰 요청 1단계 문서를 참고해 실습 환경을 구축한다.
- 이전 방탈출 예약 관리 미션 코드 위에 사용자 예약 기능을 쌓는다.
- 본인 코드 또는 페어 코드 중 시작점을 정하고, 마이그레이션한 코드는 하나의 커밋으로 합친다.
- PR 본문에는 리뷰어가 마이그레이션 커밋을 제외한 사이클 1 작업만 볼 수 있도록 리뷰 범위를 명시한다.
- 페어와 처음부터 다시 구현하지 않는다.

## 진행 방식

- 미션은 과제 진행 요구 사항, 기능 요구 사항, 프로그래밍 요구 사항으로 구성된다.
- 기능 구현 전에 기능 목록을 만들고 기능 단위로 커밋한다.
- 기능 요구 사항에 기재되지 않은 내용은 스스로 판단해 구현한다.
- 사용자 예약 API의 URL 경로, HTTP 메서드, 요청/응답 형태, 상태 코드는 직접 결정한다.

## 목표

- 토론 활동에서 정한 API 설계 규칙을 코드에 적용한다.
- API 설계 결정마다 왜 그렇게 설계했는지 PR 본문에 남긴다.
- 새로 추가되는 사용자 API와 기존 관리자 API의 관계를 의식한다.

## 기능 요구사항

지난 미션까지는 관리자가 전화 또는 현장 예약을 받아 시스템에 직접 등록했다. 이번 사이클부터는 사용자가 브라우저에서 직접 예약하는 서비스로 발전시킨다. 별도 로그인은 없으며 사용자는 이름으로 식별된다.

백엔드 API 추가와 사용자가 보는 화면 구현을 함께 진행한다. API에 맞춰 클라이언트 화면을 만들고, 각 단계의 화면이 브라우저에서 정상 동작하는 것까지 확인한다.

잘못된 입력에 대한 에러 처리는 사이클 2의 주제다. 이번 사이클에서는 정상 흐름을 만드는 데 집중한다.

### 1단계: 테마 도메인 추가

- 방탈출 게임에 테마 정보를 추가한다.
- 테마는 이름, 설명, 썸네일 이미지 URL을 가진다.
- 모든 테마의 시작 시간과 소요 시간은 동일하다고 가정한다.
- 예약에 테마 정보를 포함하도록 기존 코드를 변경한다.
- 관리자가 테마를 추가하고 삭제할 수 있다.

### 2단계: 사용자 예약

- 사용자가 날짜와 테마를 선택하면 예약 가능한 시간 목록이 표시된다.
- 예약 가능한 시간이란 관리자가 등록한 시간 중 해당 날짜와 테마에 아직 예약이 없는 시간이다.
- 사용자가 예약 가능한 시간을 선택해 본인의 이름으로 예약한다.
- 같은 날짜와 시간이라도 테마가 다르면 각각 예약할 수 있다.

### 3단계: 인기 테마 조회

- 최근 1주 동안 예약이 많았던 테마 상위 10개를 조회한다.
- 예를 들어 오늘이 5월 8일이면 게임 날짜가 5월 1일부터 5월 7일까지인 예약을 집계해 인기 순서대로 10개를 응답한다.

## 프로그래밍 요구사항

- Java 코드 컨벤션을 지킨다.
- 기본적으로 Woowacourse Java Style Guide를 원칙으로 한다.
- 레벨 1에서 만든 규칙 중 좋은 코드 작성을 위해 의미 있는 규칙을 정리하고 적용한다.
- 이번 사이클에서 구현한 API에 대한 요구사항 테스트를 작성한다.
- 정상 흐름인 예약 가능 시간 조회, 예약 생성, 다시 조회했을 때 해당 시간이 빠지는 흐름을 검증한다.
- `data.sql`로 테스트용 초기 예약 데이터를 충분히 넣어 인기 테마 조회 결과를 검증할 수 있게 한다.

## 과제 진행 요구사항

- 구현을 시작하기 전에 기능 요구사항을 분석해 기능 목록을 정리한다.
- `README.md`에 구현할 기능 목록과 직접 결정한 API 명세를 정리한다.
- 커밋 단위는 `README.md`에 정리한 기능 목록 단위로 나눈다.
- AngularJS Git Commit Message Conventions를 참고해 커밋 메시지를 작성한다.
- PR 본문에 주요 API 설계 결정과 이유를 남긴다.
