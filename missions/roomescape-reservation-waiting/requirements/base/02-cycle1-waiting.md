# 사이클 1: 예약 대기

## 한 줄 요약

테스트 전략을 적용하면서 이미 예약된 슬롯에 대기를 신청하고, 본인의 예약과 대기를 함께 관리할 수 있게 만든다.

## 실습 환경

- `spring-roomescape-waiting` 저장소를 기반으로 진행한다.
- 온라인 코드 리뷰 요청 1단계 문서를 참고해 실습 환경을 구축한다.
- 방탈출 사용자 예약 미션 코드 위에 예약 대기 기능을 쌓는다.
- 본인 코드 또는 페어 코드 중 시작점을 정하고, 마이그레이션한 코드는 하나의 커밋으로 합친다.
- PR 본문에는 리뷰어가 마이그레이션 커밋을 제외한 작업만 볼 수 있도록 리뷰 범위를 명시한다.
- 페어와 처음부터 다시 구현하지 않는다.

## 진행 방식

- 미션은 과제 진행 요구 사항, 기능 요구 사항, 프로그래밍 요구 사항으로 구성된다.
- 기능 구현 전에 기능 목록을 만들고 기능 단위로 커밋한다.
- 기능 요구 사항에 기재되지 않은 내용은 스스로 판단해 구현한다.
- 예약과 대기의 데이터 모델, 대기 만료 조건, 순번 표시 방식은 직접 결정한다.

## 목표

- 토론 활동에서 정한 테스트 전략을 코드에 적용한다.
- 테스트 결정마다 왜 이 방식으로 테스트했는지 PR 본문에 남긴다.
- 새로 추가하는 대기 도메인이 기존 예약 도메인과 어떤 관계인지 의식한다.

## 기능 요구사항

방탈출 사용자 예약 미션까지는 한 슬롯인 날짜, 시간, 테마에 한 명만 예약할 수 있었고 이미 예약된 슬롯은 사용자에게 보이지 않았다. 이번 사이클부터는 이미 예약된 슬롯에 대기를 신청할 수 있고, 사용자는 본인의 예약과 대기를 함께 조회할 수 있다.

백엔드 API 추가와 사용자가 보는 화면 구현을 함께 진행한다. API에 맞춰 클라이언트 화면을 만들고, 각 단계의 화면이 브라우저에서 정상 동작하는 것까지 확인한다.

### 1단계: 예약 대기 신청/취소

- 이미 다른 사용자에 의해 예약된 슬롯에 대기를 신청할 수 있다.
- 슬롯은 날짜, 시간, 테마 조합이다.
- 같은 슬롯에 대한 대기는 신청 순서대로 순번이 부여된다.
- 같은 사용자는 같은 슬롯에 중복 대기할 수 없다.
- 사용자는 본인의 대기를 취소할 수 있다.

### 2단계: 내 예약 목록 조회

- 이전 미션의 내 예약 목록 조회를 확장한다.
- 사용자의 예약과 대기가 상태로 구분되어 함께 표시된다.
- 대기에는 본인의 대기 순번도 함께 보여준다.

## 프로그래밍 요구사항

- Java 코드 컨벤션을 지킨다.
- 기본적으로 Woowacourse Java Style Guide를 원칙으로 한다.
- 레벨 1에서 만든 규칙 중 좋은 코드 작성을 위해 의미 있는 규칙을 정리하고 적용한다.
- 이번 사이클에서 구현한 API에 대한 요구사항 테스트를 작성한다.
- 토론에서 정한 테스트 전략에 맞춰 단위 테스트와 통합 테스트 범위를 결정해 테스트를 작성한다.

## 과제 진행 요구사항

- 구현을 시작하기 전에 기능 요구사항을 분석해 기능 목록을 정리한다.
- `README.md`에 구현할 기능 목록과 직접 결정한 API 명세를 정리한다.
- 커밋 단위는 `README.md`에 정리한 기능 목록 단위로 나눈다.
- AngularJS Git Commit Message Conventions를 참고해 커밋 메시지를 작성한다.
- PR 본문에 예약과 대기의 데이터 모델 결정, 테스트 전략 결정과 각각의 이유를 남긴다.

## 미션 중 기록

- 토론 활동에서 정한 테스트 전략을 의식하며 코드 작성
- 레벨 1 방식의 테스트가 통하지 않는 순간
- 페어와 설계 또는 테스트 의견이 갈렸던 순간과 합의 과정
- 규칙을 적용해서 변경한 코드 1곳 이상
- 테스트 작성이 어려웠던 코드 1곳 이상
- 막힌 순간 1회 이상

- 규칙에 의한 코드 변경 1회 이상
- 미션 중 기록 작성
