## Purpose

This case tests whether `spring-usecase-implementation` can produce or safely refuse backend work when the prompt is at `L5` quality.

The prompt quality changes by level, but the hidden oracle remains strict. A low-quality prompt is allowed to be incomplete; the evaluator still checks backend behavior, architecture, tests, and verification against the oracle.

## Implementation Prompt Payload

Use only this section as the implementation prompt for prompt-sufficiency baseline runs.

Implement reservation waiting creation and rank calculation. Members can wait only for an already reserved date/time/theme slot, cannot wait on their own reservation, and cannot create duplicate waiting for the same slot. Waiting rank must be calculated by request order with a deterministic tie-breaker. My reservations should include both reservations and waiting entries, and waiting entries should show the current rank. Duplicate waiting must be protected by service validation and a database constraint or equivalent final guard. Add tests for occupied slot waiting, available slot rejection, own reservation rejection, duplicate waiting rejection, rank ordering, my list response, and run full verification.
