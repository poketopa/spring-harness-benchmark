## Purpose

This case tests whether `spring-usecase-implementation` can produce or safely refuse backend work when the prompt is at `L5` quality.

The prompt quality changes by level, but the hidden oracle remains strict. A low-quality prompt is allowed to be incomplete; the evaluator still checks backend behavior, architecture, tests, and verification against the oracle.

## Implementation Prompt Payload

Use only this section as the implementation prompt for prompt-sufficiency baseline runs.

동시 로그인 정책을 구현해 주세요. 같은 회원이 다시 로그인하면 새 로그인만 유효하고 이전 토큰은 더 이상 사용할 수 없어야 합니다. 다른 회원의 로그인은 영향을 받지 않아야 합니다. 서버 쪽 세션 상태나 동등한 저장 구조로 active session을 검증하고, 이전 토큰 사용은 의도한 인증 실패 응답으로 처리해 주세요. 같은 회원이 동시에 로그인하는 race에서도 DB 제약 예외가 500으로 노출되지 않아야 합니다. 새 토큰 성공, 이전 토큰 실패, 다른 회원 토큰 유지, 실제 동시 로그인 테스트를 추가하고 전체 검증을 실행해 주세요.
