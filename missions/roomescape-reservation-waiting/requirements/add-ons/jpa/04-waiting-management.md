# 4단계: 예약 대기 관리

## 목표

- 어드민의 예약 대기 관리 기능을 구현한다.
- 예약 취소 시 대기 자동 승인을 구현한다.

## 들어가기 전 자기 진단

- 자동 승인 로직을 어디에 둘 것인가?
- Service 메서드 한 줄, 별도 도메인 이벤트, 기타 방식 중 무엇을 선택할 것인가?
- 트랜잭션 경계는 어디까지 굳혀야 한다고 보는가?

## 요구사항

### 어드민 기능

- 어드민은 대기 목록을 조회할 수 있다.
- 어드민은 예약 대기를 취소할 수 있다.
- 예약 대기 승인은 수동 또는 자동 중 하나를 선택해 구현한다.

### 수동 승인

- 예약 대기 관리 페이지에서 승인 버튼을 제공한다.
- 어드민이 승인하면 해당 대기를 예약으로 전환한다.

### 자동 승인

- 예약 취소가 발생하면 우선순위가 가장 높은 다음 대기를 자동으로 예약으로 전환한다.

## 자동 승인 구현 시 관찰할 신호

- 트랜잭션 경계: `@Transactional` 한 메서드 안에서 처리할 것인지, 책임을 분리할 것인지 결정한다.
- 동시성: 같은 자리에서 두 사용자가 동시에 취소 또는 승인할 때 일관성이 유지되는지 확인한다.
- flush 순서: 영속성 컨텍스트가 `Reservation`과 `Waiting` 두 도메인을 함께 변경할 때 어떤 엔티티가 먼저 flush되는지 확인한다.

## 완료 기준

- 시간이 부족해 4단계까지 완료하지 못했다면 PR 본문에 어디까지 만들었고 무엇이 남았는지 남긴다.
- 수동 승인까지만 구현하고 자동 승인을 미완성으로 남기는 것도 선택 가능하다.
- 자동 승인을 구현한다면 트랜잭션, 동시성, flush 순서에 대한 관찰을 기록한다.
